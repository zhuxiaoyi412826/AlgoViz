B树/B+树 特有控制

原理讲解

B树是一种多路平衡查找树，每个节点最多有m-1个关键字，m个子节点。所有叶子节点在同一层。

堆 - 底层数组表示

父子关系: 左子 = 2i+1, 右子 = 2i+2, 父 = floor((i-1)/2)

B树 vs B+树 对比

特性 B树 B+树
结构 所有节点存储数据 索引节点仅存键，叶子存储全部数据
叶子节点 无需链表连接 叶子节点通过双向链表连接
查询效率 不稳定，最坏直达叶子 稳定，需到叶子层
范围查询 需中序遍历 链表顺序访问，高效
磁盘IO 较多 更少（索引节点更小）
适用场景 文件系统、NoSQL 数据库索引、MySQL
